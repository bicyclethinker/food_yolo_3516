# Food5 YOLOv5 食物识别模型说明

## 1. 这个模型是做什么的？

这个模型的作用是：**看到一张食物图片，判断图片里是哪一种食物，并给出检测框和置信度。**

当前模型可以识别 5 类食物：

| 类别编号 | 英文类别名 | 中文含义 |
|---:|---|---|
| 0 | `chicken_wings` | 鸡翅 |
| 1 | `dumplings` | 饺子 |
| 2 | `fried_rice` | 炒饭 |
| 3 | `hamburger` | 汉堡 |
| 4 | `ice_cream` | 冰淇淋 |

它目前不是一个“大而全”的食物识别模型，而是一个 **food5 小模型**。它的主要意义是：先把“训练 → 导出 ONNX → 本地推理 → 后续上板部署”这条完整技术路线跑通。


每一类大约：

```text
训练集：600 张
验证集：150 张
```

### 3.2 标签方式

当前标签采用的是快速跑通流程用的“整张图标注法”。

也就是说，每张图的标签大致是：

```text
类别编号 0.5 0.5 1.0 1.0
```

它表示：

```text
这个食物目标框在整张图片中心
宽度占整张图片
高度占整张图片
```

换句话说，现在模型学到的更像是：

```text
这整张图片属于哪种食物
```

而不是非常精确地学会：

```text
食物在图片中具体占哪一小块区域
```

所以当前模型适合作为比赛 MVP 和上板验证模型，但如果后面要做更精准的目标检测，需要手工精标一批图片。

## 5. ONNX 模型输入输出是什么意思？

本地测试 `best.onnx` 后，得到模型信息：

```text
input name : images
input shape: [1, 3, 640, 640]
input type : tensor(float)

output name : output
output shape: [1, 25200, 10]
output type : tensor(float)
```

### 5.1 输入 `[1, 3, 640, 640]` 是什么意思？

```text
[1, 3, 640, 640]
```

可以拆开理解：

| 维度 | 含义 |
|---:|---|
| 1 | 一次输入 1 张图片 |
| 3 | 图片有 3 个颜色通道，也就是 RGB |
| 640 | 图片高度为 640 |
| 640 | 图片宽度为 640 |

也就是说，模型要求输入的是一张经过处理后的图片：

```text
1 张 RGB 图片，尺寸为 640 × 640
```

图片送进模型前，需要做预处理：

1. 读取图片；
2. 按比例缩放到 640×640；
3. 不够的地方补灰边，这叫 `letterbox`；
4. BGR 转 RGB；
5. HWC 转 CHW；
6. 像素从 0~255 归一化到 0~1；
7. 增加 batch 维度，变成 `[1, 3, 640, 640]`。

---

### 5.2 输出 `[1, 25200, 10]` 是什么意思？

```text
[1, 25200, 10]
```

可以拆开理解：

| 维度 | 含义 |
|---:|---|
| 1 | 对 1 张图片的输出 |
| 25200 | 模型提出了 25200 个候选框 |
| 10 | 每个候选框有 10 个数字 |

这里的 25200 可以理解成：

```text
模型在图片上密密麻麻地猜了 25200 个可能有食物的位置
```

每个候选框的 10 个数字是：

```text
x, y, w, h, object_conf, class0, class1, class2, class3, class4
```

含义如下：

| 位置 | 名称 | 含义 |
|---:|---|---|
| 0 | `x` | 框中心点的 x 坐标 |
| 1 | `y` | 框中心点的 y 坐标 |
| 2 | `w` | 框的宽度 |
| 3 | `h` | 框的高度 |
| 4 | `object_conf` | 这个框里有没有目标的置信度 |
| 5 | `class0` | 是 `chicken_wings` 的分数 |
| 6 | `class1` | 是 `dumplings` 的分数 |
| 7 | `class2` | 是 `fried_rice` 的分数 |
| 8 | `class3` | 是 `hamburger` 的分数 |
| 9 | `class4` | 是 `ice_cream` 的分数 |

模型不会直接告诉我们“最终答案”。它只是给出大量候选框和分数。我们还要自己做后处理。

## 6. 后处理是什么？

模型输出的是一堆原始数字，不能直接给用户看。  
后处理就是把这些数字整理成人能看懂的结果。

后处理主要包括：

### 6.1 计算最终置信度

每个框有两个关键分数：

```text
object_conf：这个地方有没有目标
class_conf：这个目标属于某个类别的概率
```

最终置信度一般这样算：

```text
最终置信度 = object_conf × class_conf
```

比如：

```text
object_conf = 0.9
ice_cream 分数 = 0.8
最终置信度 = 0.9 × 0.8 = 0.72
```

意思是：模型比较确信这个框里是冰淇淋。

### 6.2 过滤低置信度框

如果某个框最终置信度太低，比如小于 `0.25`，就丢掉。

```text
conf < 0.25 → 不要
conf >= 0.25 → 保留
```

### 6.3 坐标格式转换

YOLO 输出的框是：

```text
x_center, y_center, width, height
```

画图时通常需要：

```text
x1, y1, x2, y2
```

也就是左上角和右下角坐标。

所以要做一次转换：

```text
x1 = x_center - width / 2
y1 = y_center - height / 2
x2 = x_center + width / 2
y2 = y_center + height / 2
```

### 6.4 坐标还原回原图

模型看到的是 640×640 的图片，但原图可能不是这个尺寸。

所以检测框坐标还要从 640×640 的模型输入坐标，还原回原始图片坐标。

### 6.5 NMS 去重

模型可能会对同一个食物框出好几个相近的框。

NMS 的作用就是：

```text
同一个目标附近有多个框
只保留置信度最高的那个
删除重复框
```

NMS 的全称是 Non-Maximum Suppression，中文一般叫“非极大值抑制”。
